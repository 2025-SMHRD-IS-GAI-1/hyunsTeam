★로그인 기능★

- 일반로그인
1. 유저가 이메일(이메일인증)과 비밀번호를 입력한다.
2. 이때 유저 이메일형식이 올바르지않다면 서버로 안넘김.
2. 형식이 올바르다면 유저의 이메일과 비밀번호를 비동기방식으로 서버로 넘기고 서버에서 DB조회를 실시한다 
3. 조회된 DB가 존재한다면 해당 유저의 정보를 가져올건데 
    이때 유저email과 유저name 유저연락처를 세션에 저장하고 main페이지로 redirect
4. 만약 일치하는 내용이 없다면 에러문구 출력

- 소셜로그인
1. 소셜 로그인한경우 비동기방식으로 해당 유저의 정보가 DB에 존재하는지 체크
2. 만약 존재하지않다면 회원가입창으로 변경(이때 회원가입창은 휴대폰번호만 인증하고 가입)
3. 만약 존재한다면 session에 유저정보를 저장하고 session만들고 메인창으로 넘어감.


회원가입기능
1. 로그인창에서 유저가 회원가입버튼을 누르면 회원가입창으로 넘어간다
2. 유저가 이용약관에 동의할시 다음페이지로 넘어갈수있다
3. 유저가 소셜로그인유저일경우 휴대폰 번호만 인증하도록하고 가입이 세션에서 유저정보와 휴대폰번호 소셜타입를 DB에 등록한다
4. 유저가 로컬사용자일경우 이메일입력(이메일인증), 비밀번호입력, 비밀번호2차입력, 별명, 휴대폰번호(인증)
